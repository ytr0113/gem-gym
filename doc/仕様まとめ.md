# 仕様まとめ

## システム構成 (案)
モダンで開発効率が良く、拡張性のある構成を提案するね！

- **Frontend**: Nuxt.js (Vue 3), Tailwind CSS
- **Backend / Database**: Supabase (PostgreSQL, Authentication)
- **AI Integration**: Vercel AI SDK (Gemini API) (Nuxt対応確認) または Gemini API直接利用
- **Hosting**: Vercel (推奨)

## 機能一覧
### 1. トレーニング記録 (Logging)
- **ワークアウト作成**: 日付ごとにトレーニングセッションを作成
- **種目選択**: DBから種目を選択、または新規追加
- **セット記録**: 重量(kg), レップ数(回), RPE(自覚的運動強度)の記録
- **メモ機能**: 種目ごと、またはセッション全体のメモ

### 2. データ可視化・分析 (Visualization)
- **ダッシュボード**: 直近のトレーニング、週間ボリュームなどを表示
- **グラフ表示**: 
  - 推定1RMの推移
  - 部位別ボリュームの割合
- **ボディヒートマップ**: 鍛えた部位を人体の図で可視化（やりたい！）

### 3. AI トレーニング提案 (AI Advisor)
- **Gemini連携**: 
  - 「昨日は胸をやったから、今日は背中はどう？」といった提案
  - 過去の記録に基づいた重量設定のアドバイス
  - チャット形式での相談

## データモデル (簡易)
- **Users**: ユーザー情報
- **Exercises**: 種目マスタ (名前, 対象部位, 種別)
- **Workouts**: トレーニングセッション (日付, ユーザーID)
- **WorkoutItems**: セッション内の実施種目
- **Sets**: 1セットごとの記録 (重量, レップ数, RPE)
